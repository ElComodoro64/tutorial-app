Recursos y materiales del tutorial

Esta sección recopila los recursos utilizados para elaborar el tutorial: documentación oficial consultada, herramientas empleadas y materiales propios que acompañan a la entrega (código fuente y páginas HTML).

El objetivo es dejar claro de dónde se ha obtenido la información y qué elementos acompañan al trabajo, pensando tanto en la corrección como en posibles revisiones posteriores.

Índice de la sección

  • Para asegurar que el contenido del tutorial es correcto y actualizado se ha recurrido principalmente a documentación oficial:

    • Documentación de Angular: descripción general del framework, guía de inicio, uso de Angular CLI y enrutador.
    • Documentación de Node.js y npm: instalación, comprobación de versiones y variables de entorno.
    • Referencias a TypeScript orientadas a entender la estructura de los componentes y los archivos .ts.

    Cada explicación técnica del tutorial se ha contrastado con estas fuentes para evitar errores y facilitar futuras ampliaciones.

  • El desarrollo del tutorial y de la aplicación Angular se ha realizado con un conjunto de herramientas concretas:

    • Visual Studio Code como editor principal de código.
    • Terminal de comandos para ejecutar node, npm y los comandos de Angular CLI (ng new, ng serve, ng generate).
    • Navegador web (modo desarrollador) para visualizar la aplicación y revisar la consola ante errores.
    • Sistema de archivos local para organizar capturas de pantalla, imágenes y el propio proyecto.

    Estas herramientas son fácilmente accesibles para cualquier alumno, por lo que el tutorial puede reproducirse en otros equipos sin necesidad de software específico adicional.

  • Además de la documentación consultada, la práctica incluye materiales propios que forman parte de la entrega:

    • El proyecto Angular completo del tutorial, con todas las secciones y rutas configuradas.
    • Las imágenes utilizadas para ilustrar la Bitácora (ancla, olas, botella, bitácora, puerto, etc.), guardadas en la carpeta correspondiente del proyecto.
    • Las páginas HTML de la Bitácora original (Actividad 1), usadas como base narrativa y estilo de referencia.
    • La URL del repositorio donde se publicará el código del tutorial, en caso de subirse a una plataforma como GitHub.

    Estos materiales permiten revisar tanto el resultado visible en el navegador como la implementación interna de la aplicación.

  • Dado que el trabajo puede ser analizado con herramientas automáticas, se ha procurado:

    • Citar explícitamente las ideas tomadas de documentación oficial, adaptando la redacción al estilo de la Bitácora.
    • Evitar copiar y pegar textos largos, optando por resúmenes y explicaciones propias basadas en la lectura de las fuentes.
    • Mantener una estructura original en el tutorial, tanto en la narrativa como en la organización de las secciones.

    El resultado es un trabajo apoyado en documentación fiable, pero con una voz propia coherente con la Bitácora del Comodoro.
