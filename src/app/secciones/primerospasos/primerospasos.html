Primeros pasos: nuestra primera página en Angular

Tras crear el proyecto con Angular CLI, el siguiente objetivo es conseguir que la aplicación muestre una página propia en lugar del ejemplo por defecto. Para ello, se genera una primera sección, se limpia la plantilla inicial y se configura el enrutado para que esa página sea la que se vea nada más entrar.

Los pasos que se describen a continuación están pensados para orientar a quien da sus primeros pasos con Angular, destacando no solo qué hay que hacer, sino también dónde suelen aparecer los errores más habituales.

Índice de la sección

  • Partiendo de un proyecto recién creado con ng new, el primer paso consiste en generar una página propia que sirva como punto de partida. En este tutorial se utiliza una sección llamada Justificación.

    Desde la carpeta raíz del proyecto se ejecuta:

    cd tutorial-app
  ng generate component secciones/justificacion --standalone

    Este comando crea, dentro de src/app/secciones/justificacion/, los archivos necesarios para la nueva sección:

    • justificacion.ts: definición del componente standalone.
    • justificacion.html: plantilla HTML de la página.
    • justificacion.css: estilos asociados a esta sección.

    Para comprobar que la página responde, se puede escribir un mensaje simple en justificacion.html, que actuará como “Hola Mundo” inicial: 

    <h2>Justificación del tutorial</h2>
  <p>Esta es la primera página creada con Angular en este proyecto.</p>

    Si el proyecto aún no está abierto en Visual Studio Code, puede hacerse desde la consola con code ., siempre que el editor esté instalado en el sistema.

  • Cuando se ejecuta un proyecto recién creado, Angular muestra una página de ejemplo con logotipos y enlaces que no forman parte del tutorial. Antes de avanzar, conviene sustituir esta plantilla por una estructura propia más sencilla.

    Para ello, se edita el archivo src/app/app.html y se mantiene únicamente el esqueleto que interesa. Un ejemplo básico podría ser: 

    <header>
    <h1>Bitácora Angular del Comodoro</h1>
    <!-- Aquí irá el menú de navegación -->
  </header>
  
  <main>
    <router-outlet></router-outlet>
  </main>

    El resto del contenido generado por Angular (logotipos, tarjetas de ejemplo, enlaces, etc.) puede eliminarse sin problema. Lo importante es conservar el elemento <router-outlet>, que es donde se cargarán las distintas secciones del tutorial.

    Si tras limpiar la plantilla aparece una pantalla en blanco, suele deberse a que se ha eliminado el router-outlet o a que aún no hay ninguna ruta configurada para mostrar una sección concreta.

  • Para que la sección Justificación se muestre al entrar en http://localhost:4200/, es necesario configurarla como ruta principal en el enrutador de la aplicación.

    En el archivo src/app/app.routes.ts se importa la sección Justificación y se define la ruta por defecto (path: ''): 

    import { Routes } from '@angular/router';
  import { Justificacion } from './secciones/justificacion/justificacion';
  
  export const routes: Routes = [
    {
      path: '',
      component: Justificacion
    }
  ];

    El valor path: '' indica que esta será la página que se mostrará cuando el usuario acceda a la raíz de la aplicación, sin ningún segmento adicional en la URL.

    Si existe otra ruta con path: '' apuntando a un componente distinto, esa ruta será la que tome el control. Solo debe haber una ruta de inicio clara.

  • Una vez creada la sección y configurada la ruta, se puede iniciar el servidor de desarrollo para comprobar el resultado en el navegador. Desde la carpeta del proyecto se ejecuta: 

    ng serve

    Cuando la compilación finaliza, Angular indica la dirección donde está disponible la aplicación, normalmente: 

    http://localhost:4200/

    Al acceder a esa URL, la página debería mostrar la cabecera definida en app.html y, en la zona central, el contenido de justificacion.html con el “Hola Mundo” de prueba.

    Mientras el servidor está en marcha, la consola no recupera el prompt, ya que el proceso permanece en ejecución para recompilar ante cualquier cambio. Para detenerlo y volver al estado normal de la consola, se utiliza: 

    Ctrl + C

    Si la página sigue mostrando la plantilla de ejemplo de Angular, conviene revisar que app.html ha sido actualizado y que app.routes.ts apunta realmente a la sección Justificación como ruta inicial.

  • Aunque la primera página creada haya sido Justificación, la portada del tutorial debe ser la encargada de dar la bienvenida al usuario. Para ello se genera una nueva sección Portada y se ajustan las rutas para que sea la página inicial.

    Desde la carpeta raíz del proyecto se ejecuta:

    ng generate component secciones/portada --standalone

    En src/app/secciones/portada/portada.html se puede definir un contenido sencillo de inicio, por ejemplo: 

    <h2>Portada del tutorial Angular</h2>
  <p>Bienvenido a la Bitácora del Comodoro en Angular.</p>

    A continuación, se actualiza el archivo src/app/app.routes.ts para que la ruta de inicio apunte a Portada y Justificación quede como una sección accesible desde el menú: 

    import { Routes } from '@angular/router';
  import { Portada } from './secciones/portada/portada';
  import { Justificacion } from './secciones/justificacion/justificacion';
  
  export const routes: Routes = [
    {
      path: '',
      component: Portada
    },
    {
      path: 'justificacion',
      component: Justificacion
    }
  ];

    Más adelante, el menú definido en app.html podrá incluir enlaces a estas rutas mediante routerLink, de forma que el usuario pueda navegar entre la Portada, la Justificación y el resto de secciones del tutorial.

    Con este cambio, la aplicación deja de ser una sola pantalla fija y empieza a comportarse como un pequeño cuaderno de navegación, con la portada al frente y el resto de secciones ordenadas detrás.
