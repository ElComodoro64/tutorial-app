Aplicación de ejemplo: la Bitácora Angular del Comodoro

La práctica pide desarrollar una aplicación de ejemplo con el framework. En nuestro caso, esa aplicación es precisamente este tutorial: una Single Page Application (SPA) construida con Angular que organiza el contenido en secciones, rutas y componentes reutilizables.

En esta sección se resume qué demuestra esta aplicación, cómo está construida y de qué forma cumple los requisitos de la actividad.

Índice de la sección

  • La Bitácora Angular del Comodoro no es solo un conjunto de páginas: es una pequeña aplicación completa que demuestra:

    • Navegación entre secciones mediante el enrutador de Angular.
    • Uso de componentes standalone para cada apartado del tutorial.
    • Estructura clara de proyecto generada con Angular CLI.
    • Control del DOM a través de plantillas y directivas (índices, acordeones, textos que se expanden y se colapsan).
    • Estilos compartidos que dan unidad visual a toda la aplicación.

    El resultado es una SPA sencilla, pero suficiente para mostrar cómo Angular organiza una aplicación real y no solo un “Hola Mundo”.

  • Técnicamente, la aplicación se apoya en la estructura estándar de Angular generada con ng new, combinada con comandos posteriores de Angular CLI:

    • ng new tutorial-app para crear el proyecto base.
    • ng generate component secciones/<nombre> --standalone para crear Portada, Justificación, Instalación, Primeros pasos, Utilización, Funcionamiento interno y esta Aplicación de ejemplo.
    • Archivo app.html como plantilla principal, con el encabezado del tutorial, el menú de navegación y el <router-outlet>.
    • Archivo app.routes.ts como mapa de rutas que decide qué componente se carga en cada URL.
    • Archivos .html y .css propios por sección, para separar contenido y estilo manteniendo una narrativa común.

    Cada una de estas piezas forma parte de la aplicación final, y todas se han ido construyendo paso a paso en el propio tutorial.

  • La aplicación de ejemplo se ha diseñado expresamente para cubrir los puntos solicitados en la actividad:

    • Portada / Introducción: página inicial con título, marco del tutorial y acceso al resto de secciones.
    • Justificación del framework: sección dedicada a explicar por qué Angular y qué aporta frente a otras opciones.
    • Instalación e inicio: pasos detallados para preparar Node, Angular CLI y el primer proyecto funcional.
    • Primeros pasos, utilización y funcionamiento interno: secciones prácticas que explican estructura, rutas, componentes y ciclo de desarrollo.
    • Aplicación de ejemplo: este propio tutorial, que integra todas las piezas en una SPA navegable y coherente.

    De esta forma, el profesor puede revisar tanto el código como el comportamiento en el navegador y comprobar que el framework se ha utilizado de forma real, no solo teórica.

  • Aunque la aplicación cumple los requisitos de la actividad, deja espacio para ampliaciones futuras si se quiere seguir practicando:

    • Añadir más secciones con ejemplos de componentes, servicios o peticiones HTTP simuladas.
    • Incluir una zona de ejercicios donde el alumno pueda probar código propio siguiendo la misma estructura.
    • Separar la hoja de estilos en varios ficheros para modularizar todavía más la presentación.
    • Convertir el tutorial en una pequeña “academia” de Angular, añadiendo nuevas rutas y contenidos prácticos.

    Lo importante es que la base ya está: una aplicación Angular real, navegable y coherente que sirve como ejemplo de uso del framework.
